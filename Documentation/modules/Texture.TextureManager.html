Module Texture.TextureManager

TextureManager allows to mount a texture atlas as a logical resource path.

That means that calling TextureManager.getTexture() makes transparent the load of a real texture or of a subtexture of a texture atlas.

That allows to create code that doesn't depend on how data is packed. A use case is to always call TextureManager.getTexture() and, depending on how data is packed, add only at initialization phase a list of TextureManager.mountAtlas calls.

This list of atlas resources could be moreover defined by external data

Functions

mountAtlas (mountDir, atlas)
Mount a texture atlas as logica resource at a specific path. It's possible to mount the same atlas at different paths. It's possible to mount multiple atlas at same path.

Parameters:

  • mountDir the path where to mount the atlas
  • atlas the texture atlas to be mounted
unmountAtlas (mountDir, atlas)
Unmount an atlas from given path

Parameters:

  • mountDir the path where to unmount the atlas
  • atlas the texture atlas to be unmounted
getTexture (fileName, useCache)
Returns a texture given a fileName. The texture can be a physical resource or a subtexture obtained from a texture atlas. Textures not stored in mounted atlas are loaded through Assets.getText internal call

Parameters:

  • fileName name of the texture to be retrieved
  • useCache if defined override default Assets.__defaultCacheTexture value
clearCache (texture)
Remove a texture from the cache. If texture is nil, clears the whole texture cache. It calls also Assets.clearCache method If nil the whole cache is cleared

Parameters:

  • texture texture path or concrete resource to be removed from cache.
