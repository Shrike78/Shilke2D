Module Core.EventDispatcher

EventDispatcher is the base class for all classes that dispatch events.

This is the Shilke2D version of the Flash class with the same name. Objects can communicate with each other through events. Compared the the Flash event system, Shilke2D's event system was highly simplified. They are simply dispatched at the target. As in the conventional Flash classes, display objects inherit from EventDispatcher and can thus dispatch events. It's possible to register a function or a method as event listenr of a specific event type for an object that derived from EventDispatcher

dispose ()
First it dispatches a REMOVE_FROM_JUGGLER event to be safely detatched by jugglers. Then it remove all the event listeners.
addEventListener (eventType, listenerFunc, listenerObj)
Register a function or a method as listener of a certain type of event.

Parameters:

  • eventType the type of event for which the listener is registering
  • listenerFunc function registered as callback for the event
  • listenerObj if provided, listenerFunc is considered as a method of listenerObj. If nil listenerFunc is considered as a function
removeEventListener (eventType, listenerFunc, listenerObj)
Deregister a function or a method as listener of a certain type of event.

Parameters:

  • eventType the type of event for which the stop listening
  • listenerFunc function registered as callback for the event
  • listenerObj if provided, listenerFunc is considered as a method of listenerObj. If nil listenerFunc is considered as a function
removeEventListeners (eventType)
Remove all the listeners for a specific event

Parameters:

  • eventType the type of Event for which we want to deregister all the listeners. If nil all the listeners of all types of events will be removed.

dispatchEvent (event)
Dispatch an event to all the registered listeners. While dispatching, the lists of listener could be modified by actions of a listener callback, so to avoid problem in this phase add and remove operation are queued in a specific list and then apply in the same order that were requested.

Parameters:

  • event the event that will be dispatch to all the registered listener for this type of event
