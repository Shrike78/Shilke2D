imageHitTest (i1, p1, a1, i2, p2, a2)
Checks pixel perfect overlap of 2 raw images (MOAIImage).

Depending on __USE_SIMULATION_COORDS__ the point 0,0 of the image is considered the top or the bottom left point

Parameters:

  • i1 image 1
  • p1 top/bottom left position (vec2) of image1
  • a1 alpha treshold to consider image1 pixel transparent
  • i2 image 2
  • p2 top/bottom left position (vec2) of image2
  • a2 alpha treshold to consider image2 pixel transparent
imageHitTestEx (i1, p1, a1, i2, p2, a2, rect1, rect2, rot1, rot2)
Checks pixel perfect overlap of 2 images Depending on __USE_SIMULATION_COORDS__ the point 0,0 of the image is considered the top or the bottom left point

Parameters:

  • i1 image 1
  • p1 top/bottom left position (vec2) of image1
  • a1 alpha treshold to consider image1 pixel transparent
  • i2 image 2
  • p2 top/bottom left position (vec2) of image2
  • a2 alpha treshold to consider image2 pixel transparent
  • rect1 (optional) can define a sub region over image1 (position and size)
  • rect2 (optional) can define a sub region over image2 (position and size)
  • rot1 (optional) if true rect1 must be considered rotated 90* anticlock wise
  • rot2 (optional) if true rect2 must be considered rotated 90* anticlock wise
