Module Utils.ClassEx

Class.lua Compatible with Lua 5.1 (not 5.0).

Used to implements simple object oriented logic in lua, with single inheritance and interface implementation.

Usage:

    
- class can be used to define base classes and derived classes:

A = class()
B = class(A)

b = B()
b:is_a(A) == true
class_type(b) == A -> false
class_type(b) == B -> true

- Multiple inheritance is not allowed, but it's possible to define
interfaces (always using class) and to require that a class implements them:

iC = class()
D = class(B,iC)

d = D()
d:is_a(A) = true
d:is_a(B) = true
d:is_a(iC) = false
d:is_a(D) = true
d:implements(iC) = true

- It's also possible to implements one or more interfaces without inheritance:

iE = class()
F = class(nil,iC,iE)

f = F()
f:is_a(iC) = false
f:implements(iC) = true
f:implements(iE) = true

class_type (o)
Extends the basic lua type keyword, providing a way to check if an obj is also of a given class type. class prototype (aka metatable) for class object instances

Parameters:

  • o the object to be testet

Returns:

    string/metatable the 'type' of the object, a string for basic lua types or the

Usage:

    A = class()
B = class(A)

a = A()
b = B()

class_type(a) -> A
class_type(b) -> B
class_type("test") -> 'string'
class_type({1,2,3}) -> 'table'
is_a (o, t)
Used to check if a given object is of a given type. It can be used on generic types and on class object instances NB: Even if a class object isntance is always also a table in lua, this function return false if trying to check a class object instance over a 'table' type.

Parameters:

  • o the object to be checked
  • t the type to be checked

Returns:

    bool true if the object is of the given type, false if not.

Usage:

    A = class()
B = class(A)

a = A()
b = B()

is_a(a,A) -> true
is_a(a,B) -> false

is_a(b,A) -> true
is_a(b,B) -> true

is_a(a,'table') -> false

is_a('test','string') -> true
is_a({1,2,3},'table') -> true
implements (o, t)
Used to check if a given object implements a given interface. It can be used on generic types and on class object instances NB: we assume that a basic lua object always implements only the interface of its base type

Parameters:

  • o the object to be checked
  • t the interface type to be checked

Returns:

    bool true if the object implements the given interface, false if not.

Usage:

    iA = class()
iB = class()
C = class(nil,iA,iB)

c = C()

implements(c,iA) = true
implements(c,iB) = true
implements(c,C) = true
implements("test",C) = false
implements("test",'string') = true
super (c)

    super super class of c (nil if c is a first class)
