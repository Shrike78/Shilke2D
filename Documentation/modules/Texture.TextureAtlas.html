Module Texture.TextureAtlas

A texture atlas is a collection of many smaller textures in one big image.

This class is used to access textures from such an atlas. Using a texture atlas for your textures solves two problems: avoid frequent texture switches and reduce memory consuption A texture atlas is meant as a single image subdivided into logical named regions. Once a atlas is created is possible to add new named regions, having a 1:1 between regions and subtextures. It's then possible to query for single named texure, or for a group of sorted texture that share a prefix in the name.

fromTexture (texture, regionWidth, regionHeight, margin, spacing, prefix, padding)
Automatic creation of a tilest starting from a texture, where all the regions have the same size, are packed sequentially and have no hole in the middle of the sequence. The name of each regions will be in the form of prefix%0'padding'd, where padding is optional. if not provided there will be no padding. ie: padding 3 will produce a sequence like: prefix001 prefix002 prefix003

Parameters:

  • texture can be a Texture or a path, in that case the texture is loaded using Assets.getTexture function, using defaul cache logic
  • regionWidth width of a single tile
  • regionHeight height of single tile
  • margin num of pixel on the boundary of the texture
  • spacing num of pixel between tiles
  • prefix the prefix name to be applied to each subtexture
  • padding number of ciphers to be used to enumerate subtextures

Returns:

    TextureAtlas

addRegion (name, rect, rotated, frame)
Add a new named region. Named regions are uv map rect, so x,y,w,h are in the range [0,1]

Parameters:

  • name the name of the new region
  • rect a rect with uvmap values [0,1]
  • rotated (optional) if the texture is rotated, default is false
  • frame (optional) if the texture is trimmed frame must be provided
getSortedNames (prefix)
Returns all the regions sorted by name, that begin with "prefix". If no prefix is provided it returns all the regions

Returns the number of textures with a name that starts with prefix. If no prefix is provided it returns the number of all the textures.
